# docs 协作规范

## 1. 目录职责

- 本目录存放人工维护的静态方案文档.
- 未经用户明确允许, 不要在本目录下新增文件; 优先修改已有文档或将临时复盘放入 `tmp/`.
- 实验结果, 检查报告, 图表和 summary 不放入本目录, 应放入 `reports/`.
- 训练日志, checkpoint 和生成候选不放入本目录, 应放入 `runs/`.

## 2. 写作规则

- 修改方案文档时保持章节编号连续.
- 新增长文档时文件名优先使用 `YYYYMMDD-主题.md`.
- 保持已有方案文档之间的统一口径, 不引入冲突目录结构.
- 只更新与当前任务相关的段落, 避免顺手重排全文.
- Markdown 标题最多到 H3, 命令和配置使用围栏代码块.
