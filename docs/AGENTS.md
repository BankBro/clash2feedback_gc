# docs 协作规范

## 1. 目录职责

- 本目录只存放人工维护的方向性文档, 例如总体方案, 阶段方案, 设计说明, 路线规划和 baseline 说明.
- 未经用户明确允许, 不要在本目录下新增文件; 优先修改已有文档或将临时复盘放入 `tmp/`.
- 实验结果, 阶段最终报告, 审计报告, summary, `csv`, `parquet`, `jsonl`, `trace` 等结果文件不放入本目录, 应放入 `reports/`.
- 训练日志, checkpoint, 生成候选和渲染图等运行产物不放入本目录, 应放入 `runs/`.
- 临时复盘, 交接 prompt 和网页 ChatGPT 分析上下文不放入本目录, 应放入 `tmp/`.

## 2. 写作规则

- 修改方案文档时保持章节编号连续.
- 新增长文档时文件名优先使用 `YYYYMMDD-主题.md`.
- 保持已有方案文档之间的统一口径, 不引入冲突目录结构.
- 仓库真实结果文件优先于 `docs/` 中的方案描述; 不得把未验证设想写成已完成结论, 不得臆造实验数据.
- 只更新与当前任务相关的段落, 避免顺手重排全文.
- Markdown 标题最多到 H3, 命令和配置使用围栏代码块.
