# reports 协作规范

## 1. 目录职责

- 本目录存放各阶段生成的统计表, 图, summary 和检查报告.
- 方案文档不放入本目录, 应放入 `docs/`.
- 阶段收尾或结果分析类 Markdown 可以放在对应阶段目录下, 但必须基于已生成的报告数据, 并能追溯到配置, 数据版本和脚本入口.
- 较重运行产物不放入本目录, 应放入 `runs/`.

## 2. 产物规则

- 报告按阶段目录归档, 例如 `phase0/`, `phase1_clash_detector/`.
- 报告应尽量可追溯到配置, 数据版本和脚本入口.
- 新增 Markdown 报告时, 文件名应体现阶段和用途, 例如 `phase1_final_report.md`; 不要把研究方案, 实施计划或临时讨论写入 `reports/`.
- 小型 `csv`, `json`, `md` 报告可以提交; 大型图像批量输出需先确认.
- 自动生成报告时避免覆盖人工维护文档.
