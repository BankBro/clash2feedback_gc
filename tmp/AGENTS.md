# tmp 协作规范

## 1. 目录职责

- 本目录用于保存临时材料, 中间脚本, 一次性输出和轻量复盘.
- 所有新增文件必须按日期放入对应子目录, 目录名使用 `YYYYMMDD/`, 例如 `tmp/20260510/`.
- 本目录不是正式方案文档入口; 正式方案应放入 `docs/`, 但需遵守 `docs/AGENTS.md`.
- 本目录不是正式实验报告入口; 统计表, 图表和 summary 应放入 `reports/`.

## 2. 文件保留规则

- 只保留对复现实验, 协作分析或阶段复盘有参考价值的小型文本文件.
- `.patch`, 临时日志, 草稿和一次性中间文件用完后应及时删除.
- 不提交 raw PDB/SDF, processed pkl, 大型 cache, checkpoint, runs 图片或其他大文件.
- 新增文件名优先使用 `YYYYMMDD-主题.扩展名`, 并与所在日期子目录保持一致.
